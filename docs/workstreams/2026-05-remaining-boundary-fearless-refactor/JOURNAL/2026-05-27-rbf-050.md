# RBF-050 — Chat Turn And Transport Protocol

Date: 2026-05-27

## Outcome

Completed RBF-050.

## Deletion Test

`HttpChatTransportStreamClient` was shallow in the wrong direction: it was not
large, but it still needed to know status failure cleanup, chunk projection,
replay mutation, resume preservation, caught-error policy, and terminal stream
termination. Deleting an explicit session Module would push those rules back
into the client and spread protocol behavior across execution, projection, and
error projection callers.

`DefaultChatSessionActiveTurn` passed the deletion test. It already concentrates
active subscription ownership, UI stream reader ordering, stop/dispose behavior,
prompt append timing, and completion/error transitions. A second chat turn
protocol would duplicate state instead of improving locality.

## Changes

- Added `HttpChatTransportStreamSession` as an internal Module for client-side
  HTTP transport stream protocol consumption.
- Moved status failure projection, chunk projection delegation, caught-error
  projection, terminal stopping, and reconnect-preserving error handling out of
  `HttpChatTransportStreamClient`.
- Moved `HttpChatTransportResumeStateClearer` to
  `http_chat_transport_resume_state.dart` because it is a resume lifecycle type,
  not an error projection type.
- Added session tests for status failure termination, checkpointed stream error
  recovery, and abort termination.
