# RBF-030 — Capability Descriptor Enforcement

Date: 2026-05-27

## Summary

Added a descriptor-backed capability gate in `llm_dart_provider`:

- `ProviderCapabilityGate` for provider specifications and provider objects;
- `ModelCapabilityGate` for concrete model capability profiles;
- `CapabilityGateMode.requirement` for hard runtime validation;
- `CapabilityGateMode.affordance` for UI/discovery decisions.

The registry facet resolver now uses `ProviderCapabilityGate`, so provider
model lookup and app/UI callers share one capability decision policy.

## Deletion Test

Deleting the gate would split descriptor interpretation back across registry
facet lookup, app helpers, and UI affordance code. The new Interface is deep
enough because it owns confidence policy, provider-object facet validation,
media type matching, and model/profile decisions behind a small API.

## Guardrails

`inferred` descriptors may be surfaced as affordances, but do not satisfy hard
requirements. `unknown` and absent descriptors are always rejected.
