# RBF-040 — Non-Text App Request Seams

Date: 2026-05-27

## Summary

Added app-facing request seams for non-text helpers:

- `EmbeddingRequest`
- `GenerateImageRequest`
- `GenerateSpeechRequest`
- `TranscribeRequest`

Existing helpers still work and now delegate through those request objects.
Each request object freezes app input, owns projection into the provider-facing
request type, and centralizes validation that previously lived in helper
functions or did not exist.

## Deletion Test

The seams pass because deleting them would spread model limit validation,
descriptor-backed optional feature checks, immutable app input handling, and
provider request projection back across individual helper functions.

The implementation avoided provider-native option projection. That remains
owned by concrete provider packages.
